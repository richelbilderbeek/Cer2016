Collecting the number of taxa from the species trees

Because all species trees have the same number of taxa, there will be one n_taxa per file.

filename <- find_path("toy_example_3.RDa")
df <- collect_species_tree_n_taxa(filename)
testit::assert(names(df) == c("n_taxa"))
testit::assert(nrow(df) == 1)
knitr::kable(df)

n_taxa

  5

To be sure, the species tree is plotted here:

file <- read_file(filename)
plot(file$species_trees_with_outgroup[[1]][[1]])

Collecting the number of taxa from the posterior

Because all posteriors have the same number of taxa as the species trees, this step does not make sense.

To be sure, the first phylogeny in the first posterior is plotted here:

all_trees <- extract_posteriors(file)[[1]][[1]]
plot(all_trees[[1]])
